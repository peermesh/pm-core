# Community Engagement Plan

## Response SLA
- First maintainer response on new issues: within 48 hours.
- High-severity bug triage decision: within 24 hours.
- Security reports: acknowledge within 24 hours via private advisory channel.

## Triage Workflow
1. `triage` label on intake.
2. Classify to `bug`, `enhancement`, `module-request`, or `question`.
3. Assign severity and reproducibility.
4. Create/attach work order if scope exceeds quick fix threshold.

## Label Taxonomy
- `triage`
- `bug`
- `enhancement`
- `module-request`
- `good first issue`
- `help wanted`
- `security`

## Weekly Cadence
- Weekly backlog review for unlabeled/unassigned issues.
- Weekly refresh of `good first issue` set.
- Monthly documentation accuracy check for onboarding/deployment docs.
