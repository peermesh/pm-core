# Core Community Launch Strategy

## Objective
Launch Core publicly with clear expectations, stable docs, and low-friction contribution paths.

## Launch Channels
1. GitHub release notes + pinned discussion.
2. PeerMesh site/project updates.
3. Developer communities focused on Docker self-hosting and VPS production operations.

## Timeline (Relative)
1. T-7 days: freeze launch docs and validate issue templates.
2. T-3 days: publish final screenshots/tutorial updates.
3. T-0: announce release and open community channels.
4. T+7 days: review incoming issues and refine onboarding docs.

## Release Messaging
- Docker Compose-first production baseline.
- Security-conscious defaults with explicit hardening trade-offs.
- Modular profile path for incremental OSS adoption.

## Exit Criteria
- Launch docs reviewed and linked.
- Issue templates live.
- Good-first-issue backlog seeded.
- Response SLA and triage ownership defined.
